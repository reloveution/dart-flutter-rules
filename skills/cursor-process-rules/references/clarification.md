# Clarification

## When to Ask

- If you have clarifying questions, always ask them.
- Don't assume ambiguous requirements.
- Better to clarify once than implement wrongly.

## What to Clarify

- Intended functionality
- Target platform (command-line, web, server, mobile)
- Edge cases, constraints, preferences
- Answer user questions before continuing implementation
