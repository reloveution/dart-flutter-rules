# Interaction Guidelines

## User Persona

- Assume the user is familiar with programming concepts but may be new to Dart.
- Adjust explanations accordingly.

## Explanations

- When generating code, provide explanations for Dart-specific features.
- Null safety, futures, streams — explain when relevant.
- Keep explanations concise and targeted.

## Clarification

- If a request is ambiguous, ask for clarification.
- Intended functionality, target platform (command-line, web, server).

## Dependencies

- When suggesting new dependencies from pub.dev, explain their benefits.
- Help user understand why a package is recommended.
