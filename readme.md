# Dart & Flutter AI Rules

> A curated, evolving collection of rules and skills for AI assistants working with Dart and Flutter — designed to make LLM behavior more predictable and effective.

---

## What's inside

| Directory | Tool | Contents |
|-----------|------|----------|
| [`rules/`](rules/) | **Cursor** — `.mdc` rules for Cursor IDE | Context rules for Dart/Flutter development (architecture, BLoC, state management, security, and more) |
| `CLAUDE.md` | **Claude Code** — project memory for Claude | Same ruleset adapted as a `CLAUDE.md` file for Claude Code |
| [`skills/`](skills/) | **Universal** — works with any AI assistant | Reusable skill definitions for common Flutter patterns |

### Rules
`architecture` · `bloc` · `code-quality` · `code-review` · `dart-3-updates` · `data-patterns` · `dependency-injection` · `design-principles` · `error-handling` · `naming-conventions` · `performance` · `security` · `state-management` · `widget-patterns` · and more

### Skills
`flutter-architecture` · `flutter-bloc` · `flutter-testing` · `flutter-animations` · `flutter-navigation` · `flutter-networking` · `flutter-internationalization` · `dart-mocktail` · `dart-drift` · `mermaid-diagrams` · and more

## History

This ruleset has been assembled and refined over time from multiple sources:

1. **Original rules** — collected since July 2025, available in the [`03-11-25-edits`](https://github.com/reloveution/dart-flutter-rules/tree/03-11-25-edits) branch
2. **Official Dart/Flutter AI rules** — integrated from the [Flutter team's guidelines](https://docs.flutter.dev/ai/ai-rules)
3. **Evanca's Flutter AI rules** — merged from [evanca/flutter-ai-rules](https://github.com/evanca/flutter-ai-rules) *(Firebase and Mocktail sections included as-is, to be refined in future iterations)*
4. **Cursor directory** — a few lines from [cursor.directory/rules/flutter](https://cursor.directory/rules/flutter)
5. **MAD Teacher skills** — Dart/Flutter skills from [MADTeacher/mad-agents-skills](https://github.com/MADTeacher/mad-agents-skills)

---

## Goals

- Make AI assistant behavior in Dart/Flutter projects more **predictable** and **manageable**
- Adapt rules into a **skills system** for modular, composable AI workflows
- Continuously **compress and refine** rules to reduce token usage without losing coverage
