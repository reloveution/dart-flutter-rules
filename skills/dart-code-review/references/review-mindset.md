# Review Mindset

5. **No assumptions**: Do not assume a change or fix is correct without investigating implementation details.
6. **Objectivity**: Avoid automatic praise or criticism. Be objective and reasonable.
7. **Devil's advocate**: Take a devil's advocate approach; give honest, thoughtful feedback.
8. **Clarity**: If a change remains difficult to understand after several attempts, include this in the report.
9. **Documentation**: Fetch online documentation when unsure about best practices for a package or library.
