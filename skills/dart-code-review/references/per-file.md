# Per-File Checklist

## Structure (9–11)

- File is in the correct directory
- File name follows naming conventions
- Responsibility is clear; reason for change or addition is understandable

## Readability and Logic (12–14)

- Code is readable; variable, function, and class names are descriptive and consistent
- Logic is correct; no logic errors or missing edge cases
- Code is modular; no unnecessary duplication (maintainability)

## Quality (15–17)

- Errors and exceptions are handled appropriately
- No security concerns (input validation, secrets in code)
- No obvious performance issues or inefficiencies

## Documentation and Tests (18–21)

- Public APIs, complex logic, and new modules are documented
- Sufficient test coverage for new or changed logic
- Code matches project style guide and coding patterns
- Generated files: up-to-date and not manually edited
