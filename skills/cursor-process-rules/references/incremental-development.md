# Incremental Step-by-Step Development

## When to Use

- Complex tasks involving multiple files or architectural changes.
- Ensures clarity, early feedback, prevents context overload.

## Steps

1. **Analyze** — Analyze the task thoroughly; create a detailed step-by-step plan.
2. **Present** — Present the complete plan to the user for review and discussion.
3. **Implement** — Implement changes incrementally, one step at a time.
4. **Complete code** — Provide complete, ready-to-use code for each step.
5. **Wait** — Wait for user confirmation before proceeding to the next step.
6. **Focus** — Keep responses focused and concise; avoid repeating previous steps.
7. **Efficient** — Provide only what's needed for the current step.
8. **Answer** — If user asks clarifying questions, answer before continuing.
9. **Proceed** — After user applies changes and confirms, immediately proceed to next step.

## Benefits

- Clarity; early feedback; no context overload; maximized efficiency.
