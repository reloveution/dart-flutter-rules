# Pre-Review

1. **Branch type**: Current branch must be feature, bugfix, or PR/MR branch—not main or develop.
2. **Up-to-date**: Verify branch is up-to-date with target branch (main).
3. **Target and diff**: Identify target branch; list all changed, added, and deleted files.
4. **Connected components**: For every change, look up commit title and review how connected components are implemented.
