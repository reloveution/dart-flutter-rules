# When to Use Each Type

**Class** — static structure, inheritance, APIs

**Sequence** — temporal interactions, API calls, user scenarios

**Flowchart** — algorithms, business logic, conditional logic

**State** — object lifecycle, UI states, game state

**Module Structure** — Clean Architecture, DI, microservices

**Component Interaction** — API integration, microservices comm

**Data Flow** — ETL, pipelines, user journey

**ER** — database schema, data modeling

**GitFlow** — branch strategy, release workflow

# Flutter/Dart Use Cases

- BLoC state → stateDiagram-v2
- Clean Architecture → flowchart TB + subgraph
- API flows → sequenceDiagram
- Navigation logic → flowchart
- Widget lifecycle → stateDiagram
