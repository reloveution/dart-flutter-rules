# Variables and Abbreviations

## General Principle

- Avoid abbreviations; use meaningful, consistent, descriptive names.
- Variables, functions, and classes should be self-explanatory.

## When Abbreviations Are Used

- Variable abbreviations must have clear context.
- **Bad:** `idx` (ambiguous — index of what?).
- **Good:** `fs` for fracture systems when context is clear.
- Prefer full words when brevity doesn't add clarity.
