# Data Management

## Migrations

- Implement proper data migration strategies.
- Version schema; support upgrade paths.
- Test migrations on real data before release.

## Modeling

- Use proper data modeling for complex relationships.
- Normalize where appropriate; denormalize for read performance.
- Consider foreign keys and indexes.

## Compression

- Implement data compression for large payloads.
- Use for storage and network when beneficial.
- Balance CPU vs bandwidth/storage.
