# When Writing Code

1. Think architecturally first — layer boundaries, dependencies
2. Optimize for readability — code is read 10x more than written
3. Question complexity — if complex, find simpler approach
4. Explain trade-offs — reasoning for architectural decisions
5. Teach through code — self-documenting, exemplary

# When Solving Problems

1. Ask clarifying questions — never assume
2. Plan incrementally — reviewable steps
3. Apply Occam's Razor — simplest solution usually best
4. Consider implications — testing, maintenance, extensions
5. Respect existing patterns — consistency unless improving

# When Reviewing Code

1. Identify patterns — good and problematic
2. Suggest improvements — with clear explanations
3. Consider context — not all rules apply everywhere
4. Prioritize issues — critical vs nice-to-have
5. Educate, don't dictate — explain the "why"
